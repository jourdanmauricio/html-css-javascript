Movimiento en CSS

Transiciones

Propiedades utilizadas en las transiciones

Las transiciones en CSS nos permiten aplicar un cambio de estilo gradual a los elementos del documento HTML. Especificaremos el tiempo de duración para la transformación entre estilos, de esta forma podríamos utilizarlas para dar un efecto de animación.

La interpolación es el proceso de crear pasos "intermedios" que hacen una transición suave de un estado a otro.

  • transition-property
  • transition-duration
  • transition-timing
  • transition-delay
  • transtion

transition-property

Especifica qué estilos cambiar.

transition-duration

Define el tiempo que tardará en completarse una transición. Acepta unidades de tiempo, ya sea segundos (s) o milisegundos (ms) y el valor predeterminado es 0s.

transition-timing-function

Permite variar la velocidad de una transición de CSS en el transcurso de la duración de la transición.

Por default, CSS hará la transición de sus elementos a una velocidad constante (transition-timing-function: linear). Sin embargo, las transiciones lineales parecen artificiales: los objetos tienen peso y no pueden detenerse y comenzar instantáneamente. Entrar o salir de una transición puede hacer que sus transiciones sean más animadas y naturales.

  • linear: la velocidad de la animación es uniforme en todo el recorrido
  • ease: la velocidad se acelera al inicio, luego se retarda un poco y se acelera al final de nuevo
  • ease-in: la animación empieza lentamente y va aumentando progresivamente
  • ease-out: la animación empieza muy rápida y va descendiendo progresivamente
  • ease-in-out: la animación empieza y acaba lentamente, y es en la parte central del recorrido donde la velocidad es más rápida
  • steps()
  • cubic-bezier()

transition-delay

Especifica la hora de comienzo de una transición. Si no se especifica la duración de la transición, la transición comenzará inmediatamente ya que el valor predeterminado es 0s. Esta propiedad acepta unidades de tiempo, por ejemplo segundos (s) o milisegundos (ms).

Esta propiedad es útil para escalonar las transiciones, lo que se logra al establecer un retraso de transición más largo para cada elemento subsiguiente en un grupo.

transition

Es el shorthand de las propiedades anteriores.

Si deseamos utilizar más de un estilo podemos usar la palabra clave all. El problema es que todos los efectos tendrán la misma duración y el mismo delay.

El problema de utilizar la palabra clave all no es tanto por rendimiento sino que cada vez que carguemos la página no solo ejecuta todos las transiciones definidas, sino también en las propias propiedades del elemento. Entonces al momento de cargar la página veremos transiciones que no definimos, como renderizar bordes, colores, etc.

Existen propiedades que no son animables. Ejemplo: border-style. Las propiedades animables se encuentran detalladas en: listado de propiedades animables.

<div class="box"></div>

/* CSS */
.box {
  width: 200px;
  height: 200px;
  border: thick solid black;
  margin-left: auto;
  margin-right: auto;
  margin-bottom: 3rem;
}
.transitions .box {
  background-color: magenta;
  /* transition-property: background-color;
  transition-duration: 700ms;
  transition-timing-function: ease;
  transition-delay: 0.3s;
  transition: border-color 2s linear 0.2s; */
  transition: all 2s linear 250ms;
}
.transitions .box:hover {
  background-color: cyan;
  border-color: red;
  border-radius: 2rem;
}

Transformaciones

Las transformaciones incluyen rotar, torcer, escalar y desplazar en 2D o en un espacio 3D.

Transformaciones en 2D

traslación

Las funciones de translación son aquellas que realizan una transformación en la que mueven un elemento de un lugar a otro. Existen 3 funciones de traslación: translateX() y translateY() para cada eje, y la propiedad de atajo translate(), que es una mezcla de las dos anteriores en una sola función de translación.

/* .element {
  transform: translateX(20px); 
  transform: translateY(-30px);
  transform: translateX(20px) translateY(-30px);
} */

<article class="card transform-2d-translate">
  <img src="/assets/images/puesta-sol.jpg" alt="Puesta del sol">
</article>

/* CSS */
.transform-2d-translate img {
  transition: transform 1s ease-in-out;
}
.transform-2d-translate img:hover {
  /* transform: translateX(20px);
  transform: translateY(-30px);
  transform: translate(50%, 50%); */  
  transform: translate(20px, -30px);
}

Escala

Aumentan o reducen el tamaño de un elemento. Las funciones de escalado son las siguientes:

  • scaleX(fx); Reescala el elemento un número de fx veces (sólo en horizontal).
  • scaleY(fy); Reescala el elemento un número de fy veces (sólo en vertical).
  • scale(fx, fy); Propiedad de atajo de las dos anteriores (escalado simétrico).
  • scale(fx); Equivalente al anterior: scale(fx, fx).
.element {
  /* Aumenta el tamaño 100% en x */
  transform: scaleX(2); 
  /* Aumenta el tamaño 50% en y */
  transform: scaleY(.5);
  /* Aumenta el tamaño 100% en x y 50% en y */
  transform: scale(2, 1.5);
  /* Aumenta tamaño en 50% */
  transform: scale(1.5);
  /* Invierte la imagen */
  transform: scale(-1, -1);
}

<article class="card transform-2d-scale">
  <img src="/assets/images/puesta-sol.jpg" alt="Puesta del sol">
</article>

/* CSS */
.transform-2d-scale img {
  transition: transform 2s ease-in-out;
}
.transform-2d-scale img:hover {
  /* transform: scaleX(2); 
  transform: scaleY(1);
  transform: scale(2, 1); */
  transform: scale(1);
}

Rotación

Define una transformación que gira un elemento alrededor de un punto fijo (origen de transformación, se puede personalizar utilizando la propiedad transform-origin) en un plano 2D sin deformarlo.

/* Genera rotación en eje Y. */
transform: rotateY(60deg);
/* Genera rotación en eje Z. */
transform: rotateZ(60deg);

  /* transform: rotateY(60deg); */

Torcer

Establece un ángulo para torcer, tumbar o inclinar un elemento en 2D.

  • skewX(xdeg): Establece un ángulo de xdeg para una deformación 2D respecto al eje X
  • skewY(ydeg): Establece un ángulo de ydeg para una deformación 2D respecto al eje Y
transform: skewX(20deg);
transform: skewY(20deg);
transform: skew(20deg, 20deg);

>article class="card transform-2d-skew">
  >img src="/assets/images/puesta-sol.jpg" alt="Puesta del sol">
>/article>

/* CSS */
.transform-2d-skew img {
  transition: transform 2s ease-in-out;
}
.transform-2d-skew img:hover {
  /* transform: skewX(20deg); */
  /* transform: skewY(20deg); */
  /* transform: skewY(-20deg); */
  /* transform: skewX(-20deg); */
  transform: skew(20deg, 20deg);
}
Aunque la función skew() existe, no debería ser utilizada, ya que está marcada como obsoleta y serán retiradas de los navegadores en el futuro. En su lugar deberían utilizarse skewX() o skewY(), ya sea individualmente por separado o combinadas separando por espacio.

Origen de Transformación (transform-origin)

Permite modificar el origen de las transformaciones de un elemento. Por ejemplo, el transform-origin de la función rotate() es el centro de rotación.

De forma predeterminada, el origen de una transformación es "50% 50%", que está exactamente en el centro de cualquier elemento.

Los valores pueden ser longitudes, porcentajes o las palabras clave superior, izquierda, derecha, inferior y central.

El primer valor es la posición horizontal, el segundo valor es la vertical y el tercer valor representa la posición en el eje Z. El tercer valor solo funcionará si está utilizando transformaciones 3D y no puede ser un porcentaje.

<article class="card transform-3d-multiple">
  <img src="/assets/images/puesta-sol.jpg" alt="Puesta del sol">
</article>

/* CSS */        
.transform-3d-origin img {
  transition: transform 2s ease-in-out;
  transform-origin: 0 0;
}
.transform-3d-origin img:hover {
  transform: rotate(360deg);
}

Flip Cards con transiciones y transformaciones

La propiedad backface-visibility determina si la cara posterior de un elemento es visible. La cara posterior de un elemento siempre es un fondo transparente, cuando es visible se muestra una imagen de espejo de la cara frontal.

Hay casos en los que no queremos que la cara frontal de un elemento sea visible a través de la cara posterior, como cuando se hace el efecto de voltear una tarjeta.

Esta propiedad no tiene efecto en transformaciones 2D, pues estos no tienen perspectiva.

Esta propiedad puede tener dos valores:

  • visible: la cara posterior es visible, permitiendo que la cara frontal se muestre invertida
  • hidden: la cara posterior no es visible, ocultando la cara frontal

La propiedad transform-style establece si el elemento hijo esta posicionado en el espacio 3D (preserve-3d) o esta integrado(flat) en el plano del elemento.

  • flat: Indica que el elemento hijo yace en el plano del elemento mismo
  • preserve-3d: Indica que el elemento hijo debe ser posicionado en el espacio 3D

Animaciones

Las animaciones CSS3 permiten animar la transición entre un estilo CSS y otro. Las animaciones constan de dos componentes: un estilo (que describe la animación CSS) y un conjunto de fotogramas (que indican su estado inicial y final), así como posibles puntos intermedios en la misma.

Las animaciones CSS tienen tres ventajas principales sobre las técnicas tradicionales de animación basada en scripts:

  • Son muy fáciles de usar para animaciones sencillas, puedes hacerlo incluso sin tener conocimientos de Javascript
  • La animación se muestra correctamente, incluso en equipos poco potentes. Animaciones simples realizadas en Javascript pueden verse mal (a menos que estén muy bien hechas). El motor de renderizado puede usar técnicas de optimización como el "frame-skipping" u otras técnicas para que la ejecución de la animación se vea tan suave como sea posible
  • Al ser el navegador quien controle la secuencia de la animación, permitimos que optimice el rendimiento y eficiencia de la misma, por ejemplo, reduciendo la frecuencia de actualización de la animación ejecutándola en pestañas que no estén visibles

Las subpropiedades de animation son:

  • animation-delay: Tiempo de retardo entre el momento en que el elemento se carga y el comienzo de la secuencia de la animación
  • animation-direction: Indica si la animación debe retroceder hasta el fotograma de inicio al finalizar la secuencia o si debe comenzar desde el principio al llegar al final
  • animation-duration: Indica la cantidad de tiempo que la animación consume en completar su ciclo (duración)
  • animation-iteration-count: El número de veces que se repite. Podemos indicar infinite para repetir la animación indefinidamente
  • animation-name: Especifica el nombre de la regla @keyframes que describe los fotogramas de la animación
  • animation-play-state: Permite pausar y reanudar la secuencia de la animación
  • animation-timing-function: Indica el ritmo de la animación, es decir, como se muestran los fotogramas de la animación, estableciendo curvas de aceleración
  • animation-fill-mode: Especifica qué valores tendrán las propiedades después de finalizar la animación (los de antes de ejecutarla, los del último fotograma de la animación o ambos)

Uno los valores para animation-timing-function es cubic-bezier. En la página cubic-bezier podemos ver los distintos efectos.

<article class="card my-animation">
  <img src="/assets/images/puesta-sol.jpg" alt="Puesta del sol">
</article>

/* CSS */
@keyframes myAwesomeAnimation {
  from {
    opacity: 0;
    transform: translateX(0);
  }
  to {
    opacity: 1;
    transform: translateX(100%);
  }
}

.my-animation {
  /* animation: name duration timing-function delay iteration-count direction fill-mode; */
  animation-name: myAwesomeAnimation;
  animation-duration: 2s;
  /* animation-timing-function: cubic-bezier(0, 0.97, 1, -0.02); */
  animation-timing-function: ease-in-out;
  animation-delay: 1s;
  /* animation-iteration-count: infinite; */
  animation-iteration-count: 2;
  animation-direction: normal;
  /* animation-direction: reverse; */
  /* animation-direction: alternate; */
  /* animation-direction: alternate-reverse; */
  /* animation-fill-mode: none; */
  /* animation-fill-mode: forwards; */
  /* animation-fill-mode: backwards; */
  animation-fill-mode: both;
  /* animation-play-state: running; */
  /* animation-play-state: paused; */
  animation: myAwesomeAnimation 1.5s ease 250ms 6 alternate both running;
}
