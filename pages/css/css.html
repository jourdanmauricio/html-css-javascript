Introducción

CSS provine de Cascading Style Sheets que se traduce como hojas de estilo en cascada por la forma en que CSS aplica los estilos. Es un leguaje de definición de presentación de los documentos HTML.

Así como HTML determina cuál es el contido de nuestras páginas, CSS se encarga de la cisualización de ese contenido. Se encarga del posicionamiento, de las animaciones, de los colores, transiciones, etc.

CSS no es un lenguaje de programación pero comienza a integrar características de los lengiajes de programción como las custom properties, variables, etc.

Sintaxis básica

Para definir los estilos CSS utiliza reglas. Cada regla consta de dos partes:

  • Selector arrow_right_alt es el elemento HTML (cualquier etiqueta) al que le aplicaremos estilos. El selector más básico son las etiquetas HTML, pero existen otros tipos de selectores que veremos más adelante como las clases (a través del atributo class), identificadores (a través del atributo id), selectores de atributos, selectores de hermanos, adyacentes, etc.
  • Bloque de declaración arrow_right_alt se refiere a las propiedaes que vamos a modificar. La declaración se encierra entre llaves {}, y la sintaxis para cada propiedad es propiedad: valor;. El punto y coma final indica que terminamos con la definición de esa propiedad y pasamos a la siguiente. 
    h1 {
  color: orange-red;
  background-color: skyblue;
}

    • En el ejemplo el selector es la etiqueta h1, el bloque de declaraciones se encuentra entre las llaves {} y posee dos atributos: el color del texto y el color de fondo.

Comentarios

Las líneas comentadas serán ignoradas y nos sirven para realizar aclaraciones sobre nuestro código.

<!-- comentario HTML -->
/* comentario SCC */

El atajo para realizar comentarios en Visual Studio Code se realiza presionando (Control + k + c) y para descomentar (Control + k + u). Esto puede variar de acuerdo a la configuración que podemos ver presionando (Ctrl + Shift + p) la paleta de comandos y escribiendo "comment"

Incorporando CSS

Existen tres formas de aplicar estilos CSS:

  • Etiqueta style dentro del elemento HTML arrow_right_alt En elsiguiente ejemplo se aplica un fondo amarillo y color de letra roja: 
    <div style="background-color:pink; color:red"> 
/* Este es un bloque DIV. Pintamos el fondo de amarillo para que se vea su magnitud real. Vemos como ocupa el 100% del ancho aunque no lo necesite. */
</div>
  • Etiqueta style dentro de la cabecera HEAD del documento HTML arrow_right_alt El siguiente ejemplo pintará todos los títulos de tipo h2 de color azul: 
    <!DOCTYPE html>
  <html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
      h1 {
        color: blue;
      }
    </style>
  </head>
  <body>
  </body>
</html>
  • Utilizando una hoja de estilos arrow_right_alt Es la forma correcta de incluir los estilos. En el HEAD linkeamos el archivo con extensión .css que contiene los estilos, y ya estarán disponibles para utilizarlos. Utilizamos la etiqueta link. De esta manera separamos el html de CSS. 
    <!DOCTYPE html>
  <html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
  </body>
</html>
    Y el archivo style.css: 
      h1 {
    color: red;
  }

Dentro de una hoja de estilos podemos invocar otra hoja de estilos a través de la sentencia @import. Pero es una mala práctica porque en aplicaciones grandes, al cargar la página descargamos todos los recursos bloqueando el contenido. Las sentencias @import son bloqueantes, hasta que no termine su descarga el navegador no continúa renderizando la página. En cambio, con etiquetas Link el navegador descarga el archivo en segundo plano y continua leyendo el arhivo HTML.

@import url("otra-styles.css");

Selectores básicos: Etiquetas, Identificadores y Clases

Existen 3 selectores básicos

  • Etiquetas arrow_right_alt se asocian con elementos HTML. Cuando aplicamos estilos a una etiqueta impactará sobre todas las etiquetas de ese tipo del documento. 
    <!-- html -->
<p>Lorem ipsum dolor sit amet consectetur, adipisicing elit. Tenetur enim quas ut officia quidem minus sapiente quibusdam consequatur quaerat facere.<p>

/* CSS */
p {
  color: blue;
}
  • Identificadores arrow_right_alt en los elementos HTML debemos agregar el atributo id y en css lo indentificamos por el símbolo numeral (#). En este caso el estilo solo se aplicará sobre el elemento HTML identificado por ese id. Es una mala práctica utilizar los id para establecer los estilos. Dentro de los documentos, los identificadores son únicos por lo que no estaríamos reutilizando los estilos en varios elementos. Por otro lado, más adelante veremos el algoritmo de CSS y cómo se trata la especificidad de selectores. Allí veremos porqué es un anti patrón utilizar los id para los estilos. Los id se utilizan más en relación a JavaScript. 
    <!-- html -->            
<p id="parrafo-dos">Lorem ipsum dolor sit amet consectetur, adipisicing elit. Tenetur enim quas ut officia quidem minus sapiente quibusdam consequatur quaerat facere.<p>

/* CSS */
#parrafo-dos {
  background-color: pink;
}

    Lorem ipsum dolor sit amet consectetur, adipisicing elit. Tenetur enim quas ut officia quidem minus sapiente quibusdam consequatur quaerat facere.

  • Clases arrow_right_alt en los elementos HTML debemos agregar el atributo class y en css lo indentificamos por el símbolo del punto (.). Las clases nos permiten reutilizar código, se aplicarán a todos los elementos que posean la clase asignada. Una de las ventajas es que en el documento HTML podemos agregar más de una clase a un elemento. Solo se separan las clases que le queremos asignar por un espacio.

    Como las clases son reutilizables tenemos que darles nombres alusivos a las acciones que realizan.

    <!-- html -->
<p class="text-32 bg-skyblue">Lorem ipsum dolor sit amet consectetur, adipisicing elit. Tenetur enim quas ut officia quidem minus sapiente quibusdam consequatur quaerat facere.<p>

/* CSS */
.text-32 {
  font-size: 32px;
}

.bg-skyblue {
  background-color: skyblue;
}

    Lorem ipsum dolor sit amet consectetur, adipisicing elit. Tenetur enim quas ut officia quidem minus sapiente quibusdam consequatur quaerat facere.

Selectores avanzados

Selectores de hijos

Existen dos tipos de selectores de tipo hijos.

Selector de hijos directos

Un hijo directo es un elemento que se encuentra en el primer nivel del elemento referenciado con la clase. En el siguiente ejemplo las etiquetas li son hijas directas del ul pero las etiquetas b, a pesar de ser hijas, no son hijas directas. Son "nietas". En CSS la nomenclatura para indicar hijo directo es el símbolo mayor (>). 

<ul class="hijos-directos">
  <li><b>Primavera</b></li>
  <li><b>Verano</b></li>
  <li><b>Otoño</b></li>
  <li><b>Invierno</b></li>
</ul>

/* CSS */
.hijos-directos > li {
  background-color: thistle;
}

/* no aplica porque b no es hijo directo de ul */
.hijos-directos > b {
  background-color: thistle;
}
  • Primavera
  • Verano
  • Otoño
  • Invierno

Selector de hijos descendientes

En CSS, el selector de hijo descendiente se representa por un espacio en blanco y aplica a todos los hijos del elemento si considerar el orden en el que se encuentren. No importa si son directos o no.

<ul class="hijos-descendientes">
  <li><b>Primavera</b></li>
  <li><b>Verano</b></li>
  <li><b>Otoño</b></li>
  <li><b>Invierno</b></li>
</ul>

/* CSS */
.hijos-descendientes b {
  background-color: thistle;
}
  • Primavera
  • Verano
  • Otoño
  • Invierno
El fondo se pinta distinto porque en el primer caso aplica sobre los li (elementos de bloque), mientras que en el segundo caso aplica sobre las b (elementos en línea).

Selectores de hermanos

Existen dos tipos de selectores de tipo hijos.

Los selectores de hermanos aplicarán sobre los elementos hermanos (mismo nivel) pero que se encuentren por debajo del elemento de referencia.

Selector de hermanos (en general)

El selector de hermanos en general se representa por el símbolo virgulilla (~) y aplicará a los elementos espcificados que se encuentren en el mismo nivel. En el siguiente ejemplo aplicará sobre todos los li que se encuentren en el mismo nivel que el elemento que posee la clase que referenciamos.

<ul>
  <li><b>Primavera</b></li>
  <li class="hermanos-general"><b>Verano</b></li>
  <li><b>Otoño</b></li>
  <li><b>Invierno</b></li>
</ul>

/* CSS */
.hermanos-general ~ li {
  background-color: thistle;
}
  • Primavera
  • Verano
  • Otoño
  • Invierno

¿Por qué no aplica al elemento li Primavera? Porque los selectores de hermanos aplican a elementos que sean hermanos pero que se encuentren por debajo de él. No aplican hacia arriba.

Selector hermano adyacente

El selector hermano adyacente se representa por el símbolo suma (+) y aplicará al siguiente elemento que se encuentren en el mismo nivel del elemento referenciado. En el siguiente ejemplo aplicará sobre el li Otoño.

<ul>
  <li><b>Primavera</b></li>
  <li class="hermanos-adyacentes"><b>Verano</b></li>
  <li><b>Otoño</b></li>
  <li><b>Invierno</b></li>
</ul>

/* CSS */
.hermanos-adyacentes + li {
  background-color: thistle;
}
  • Primavera
  • Verano
  • Otoño
  • Invierno

Selectores de atributos

Nos permite aplicar selectores a través de cualquier atributo relacionado a etiquetas HTML. Se representan a través de los corechetes [].

<nav class="seletores-atributos">
  <ul>
    <li><a href="#" target="_blank">Inicio</a></li>
    <li><a href="#" target="_blank">Cursos</a></li>
    <li><a href="#" target="_blank">Blog</a></li>
    <li><a href="#" target="_blank">CV</a></li>
    <li><a href="#" target="_blank">Ahora</a></li>
    <li><a href="#" target="_blank">Notas</a></li>
  </ul>
</nav>

/* CSS */
.seletores-atributos a[href] {
  color: orangered;
}
  • Inicio
  • Cursos
  • Blog
  • CV
  • Ahora
  • Notas

Aplica a todos los elementos a dentro del elemento que contenga la clase "selectores-atributos" y que posean el atributo "href". También podemos agregar el valor que posee el atributo para realizar la selección. Ejemplo:

/* CSS */
.seletores-atributos a[target="_blank"] {
  font-weight: bold;
}

Otra característica de los selectores de atributos es que pueden utilizar comodines

  • * arrow_right_alt el valor del atributo contiene los caracteres que se detallen
  • ^ arrow_right_alt el valor del atributo inicia con los caracteres que se detallen
  • $ arrow_right_alt el valor del atributo finaliza con los caracteres que se detallen
  • ~ arrow_right_alt el valor del atributo contiene el valor y se encuentra en una lista separada por espacios
  • | arrow_right_alt el valor del atributo contiene el valor y se encuentra en una lista separada por espacios y que después del texto contenga un guión medio
/* CSS */
.seletores-atributos a[target*="bl"] {
  font-weight: bold;
}
.seletores-atributos a[target^="_bl"] {
  font-weight: bold;
}
.seletores-atributos a[target$="anc"] {
  font-weight: bold;
}
.seletores-atributos a[class~="boton"] {
  font-weight: bold;
}
.seletores-atributos a[class|="menu"] {
  font-weight: bold;
}

Selector universal

Se representa por el asteristo (*) y aplica a todos los elementos HTML que existan en el documento.

* {
  font-family: sans-serif;
}

Aplica la tipografía sans-serif a todos los elementos, y al ser una tipografía estandar existe en todos los sistemas operativos (Mac, Windows o Linux).

También se puede aplicar el selector universal pero contextualmente. Aplicará a todos los elementos que se encuentren dentro del elemento referenciado.

<ul class="selector">
  <li><b>Primavera</b></li>
  <li><b>Verano</b></li>
  <li><b>Otoño</b></li>
  <li><b>Invierno</b></li>
</ul>

/* CSS */
.selector * {
  background-color: thistle;
}

Pseudoclases

Las pseudoclases aplicarán los estilos dependiendo del contexto, de la posición del elemento o de ciertos estados que tenga elemento HTML. Las pseudoclases se representan a través de los dos puntos (:).

Documentación de las distintas pseudoclases que podemos encontrar: Pseudoclases

Pseudoclases para elementos de formularios

  • required arrow_right_alt representa cualquier elemento input, select o textarea que tenga el atributo requerido establecido.
  • valid arrow_right_alt representa cualquier elemento input u otro del form cuyo contenido se valide correctamente. Esto permite hacer que los campos válidos adopten una apariencia que ayude al usuario a confirmar que su dato se ingresó correctamente.
  • invalid arrow_right_alt representa cualquier fieldset, input u otro elemento del form cuyo contenido no se valide correctamente. Esta pseudoclase es útil para resaltar errores para el usuario.
  • focus arrow_right_alt representa un elemento (como una entrada de formulario) que ha recibido el foco. Por lo general, se activa cuando el usuario hace clic o toca un elemento o lo selecciona con la tecla tabulador del teclado.
  • disabled arrow_right_alt representa cualquier elemento deshabilitado. Un elemento está deshabilitado si no se puede activar (seleccionar, hacer clic, escribir, etc.) o aceptar el enfoque.
<form class="form-pseudoclase" action="#">
  <input type="text" name="nombre" placeholder="Escribe tu nombre" required>
  <br>
  <input type="email" name="correo" placeholder="Escribe tu correo" required>
  <br>
  <input type="submit">
</form>

  border-color: yellow;
}

  border-color: green;
}

  border-color: red;
}

  background-color: skyblue;
}

  background-color: pink;
}



Pseudoclases para elementos de enlace

  • hover arrow_right_alt aplica sobre cualquier elemento HTML. Se activa cuando el usuario pasa el cursor sobre un elemento (puntero del mouse). Los estilos definidos por la pseudoclase :hover serán anulados por cualquier pseudoclase posterior relacionada con el enlace (:enlace, :visitado o :activo) que tenga al menos la misma especificidad. Para aplicar un estilo adecuado a los enlaces, coloque la regla :hover después de las reglas :link y :visited, pero antes de :active
  • visited arrow_right_alt enlaces que el usuario ya ha visitado
  • active arrow_right_alt representa un elemento (como un botón) que está siendo activado por el usuario. Cuando se usa un mouse, la "activación" generalmente comienza cuando el usuario presiona el botón principal del mouse
  • link arrow_right_alt representa un elemento que aún no ha sido visitado. Coincide con todos los elementos a no visitados que tienen un atributo href
  • target arrow_right_alt representa un elemento único (el elemento objetivo) con una identificación que coincide con el fragmento de la URL. Cuando la url actual posee el símbolo numeral (#) y se establece la pseudoclase :target se aplicarán los estilos a ese identificador 
    /* Selecciona un elemento con un ID que coincida con el fragmento de la URL actual */
:target {
  border: 2px solid black;
}
De acuerdo a la especificidad de las pseudoclases, deberían definirse en con el siguiente orden:
  1. link
  2. active
  3. visited
  4. hover

  <a href="#seccion3">Seccion 3</a>
  <a href="#seccion4">Seccion 4</a>
</nav>

Pseudoclases por posición y tipo

Las pseudoclases por posición hacen referencia a los elementos en relación a la posición que ocupan dentro de su contenedor.

  • first-child arrow_right_alt aplicará al primer elemento que tenga que sea del tipo indicado
  • last-child arrow_right_alt aplicará al último elemento que tenga que sea del tipo indicado
  • nth-child(nro) arrow_right_alt aplicará al elemento nro que tenga que sea del tipo indicado. Esta pseudoclase admite determinadas fórmulas
    • nth-child(4n) arrow_right_alt aplicará al elemento que tenga que sea del tipo indicado pero cada 4 elementos. Aplicará al 4, 8, 12, etc
    • nth-child(3n+1) arrow_right_alt aplicará al elemento que tenga que sea del tipo indicado pero cada 3 elementos + 1. Aplicará al 4, 7, 10, etc
    • nth-child(even) arrow_right_alt aplicará a los elementos pares que sean del tipo indicado
    • nth-child(odd) arrow_right_alt aplicará a los elementos impares que sean del tipo indicado

Pseudoclases por posición y tipo

Este tipo de pseudoclases no solo aplican por posición sino que también consideran el tipo de etiqueta.

  • first-of-type arrow_right_alt buscará dentro de la clase el primer tipo que coincida con el especificado

Pseudoclases de negación

Podemos utilizar esta pseudoclase cuando queremos aplicar a todos los elementos con excepción de un tipo. Si necesitamos que sean más de un elemento a exluir podemos separarlos por coma.

.articulo-pseudoclases *:not(p, boton, .clase) {
  background-color: beige;
}

Pseudoelementos

Las pseudoelementos aplicarán los estilos a partes específicas de un elemento y se representan a través de los dos dos puntos (::).

Documentación de los distintos pseudoelememtos que podemos encontrar: Pseudoelememtos

  • after y before arrow_right_alt nos permiten agregar contenido antes o después de un elemento en particular
  • first-letter arrow_right_alt los estilos se aplicarán al primer caracter
  • first-line arrow_right_alt los estilos se aplicarán a la primera línea
  • selection arrow_right_alt los estilos se aplicarán al momento de seleccionar el elemento
  • placeholder arrow_right_alt los estilos se aplicarán al atributo placeholder del elemento input refernciado

El Algoritmo de CSS

El algoritmo de CSS: es la forma en que el navegador aplica los estilos la documeto HTML. Podemos dividir el tema en tres

  • La cascada
  • La especificidad
  • La herencia

La cascada

Mecanismo que tiene el navegador para aplicar los estilos. Considera tres aspectos.

  1. Origen del código.
    • User-agent. Son los estilos que por defecto aplica el navegador a las etiquetas. Si observamos el inpector de elementos vemos estilos que no definimos pero que el navegador carga "por defecto". Para abrir el inspector de elementos podemos presionar la tecla F12 o con el botón derecho del mouse seleccionar Inspeccionar. Como vemos en la imagen el navegador aplicó estilos al encabezado h3 y a la derecha nos indica "Hoja de estilo del usuario-agente"
    • Configuración del usuario sobre el navegador o sistema operativo. El usuario puede configurar que el navegador muestre los textos un 25% más grande. Entonces despúes de los estilos del user-agent, el navegador aplica las configuraciones particulares del usuario
    • Código CSS - Estilos del autor. Finalmente el navegador aplica el código CSS relacionado al documento HTML.
  2. La especificidad del selector. En el siguiente capítulo veremos la especificidad en profundidad
  3. Orden de aparición

Para especificar el orden en que el navegador aplica los estilos podemos ver un ejemplo de un documento que solo posee un elemento encabezado de tipo h1 dentro de su body. Consideraciones sobre los estilos:

  • El h1 posee un estilo en línea asignando el colo de texto a violet
  • El documento posee estilos integrados en el head indicando que el color del h1 debe ser orange
  • El documento incorpora estilos desde una hoja de estilos donde se indica que el color del h1 debe ser skyblue

¿De qué color será el h1?Veamos la respuesta en la solopa estilos del inspector de elementos.

El color será violet porque como vemos (comenzando desde abajo) en la solapa estilos del inspector de elementos, el navegador carga sus estilos (usuario-agente. En este caso no asigna color al h1), luego aplica los estilos de la hoja de estilos externa, llamada styles2.css, donde se indica que el color debe ser skyblue. El inspector nos indica que la sentencia proviene de la línea 1 del archivo llamado styles2.css.

Al leer en head de nuestro documeto HTML aplica el color orange. En el inspector vemos que tacha el color anterior (skybkue) y genera una nueva sección donde nos indica que la sentencia corresponde a la línea 11 del archivo index2.html.

Finalmente, al pintar el elemento h1 aplica el color violeta. En el inspector tacha el color anterior (orange) y genera una nueva sección llamada element.style (estilos en línea) con el color violet.

Conclusión: el navegador aplicó los estilos en el siguietne orden:

  1. Estilos del navegador: user-agent
  2. Estilos externos vinculados a través de la etiqueta link
  3. Estilos internos definidos en la etiqueta head de nuestro documento
  4. Estilos en línea definidos dentro de la etiqueta

También podemos ver un resumen en la solapa calculados del inspector.

La especificidad

La especificidad es el peso que tiene un selector cuando ocurren conflictos de estilos. El navegador asigna mayor importancia a la especifidad que a la cascada. Cada selector posee un puntaje (peso específico) y el navegador aplicará el que mayor peso tenga sobreescribiendo los estilos anteriores.

Suponiendo que tenemos dos definiciones de estilo para un elemento h1. Primero utilizando el id y luego utilizando una clase. Lógicamente pensaremos que se aplicará el estilo de la clase porque es el último que encontró en navegador, pero como el peso de los identificadores es mayor al peso de las clases se aplicará el estilo definido a través del id dejando sin efecto la regla de la cascada.

El navegador calcula los pesos utilizando la siguiente tabla.

Peso de los selectores
Selector Valor
Etiquetas y pseudoelementos 0,0,0,1
Clases, atributos y pseudoclases 0,0,1,0
Identificadores 0,1,0,0
Estilos en línea 1,0,0,0
!Important Rompe la especificidad

Si el estilo se aplica a través de una etiqueta y una clase se sumarán los valores.

/* caso 1 */
.container h1 {
  color: red;
}
/* caso 2 */
.form-agrupar input[type="email"] {
  color: orange;
}

En el primer caso del ejemplo anterior encontramos el selector de clase (.container) y el de etiqueta (h1). El peso resultante de la regla es:

0,0,1,0 (clase) + 0,0,0,1 (etiqueta) = 0,0,1,1

En el segundo caso encontramos el selector de clase (.form-agrupar), el de etiqueta (input) y el de atributo ([type="email"]). El peso resultante de la regla es:

0,0,1,0 (clase) + 0,0,0,1 (etiqueta) + 0,0,1,0 (atributo) = 0,0,2,1

Recomendaciones:

  • Evitar la utilización de identificadores para aplicar estilos: si asignamos estilos a través de identificadores no podremos retulizar el código css porque cada identificador es único. Y por otro lado, el peso de los identificadores es superior al de las clases por lo que ante situaciones de conflicto no podremos desempatar por medio de clases.
  • Evitar el uso de !important: agregando la palabra !important a la declaración de la regla rompemos la especificidad y asignamos la mayor prioridad posible. Pero es una mala práctica porque ya no podremos sobreescribir el estilo. En ocasiones definimos un estilo y no se aplica, y en lugar de analizar en profundidad la espeficidad para analizar dónde debemos realizar el cambio, colocamos !important. Es más fácil pero es una mala práctica.
  • Maquetar mediante clases y evitar utilizar más de dos niveles.

La herencia

Es la capacidad de un selector de obtener (heredar) los valores de sus ancestros. Para aplicarla se utiliza el valor inherit y para evitarla podemos asignar otro valor (sobreescribiendo) o colocando el valor initial (que establece el valor por defecto).

Hay propiedades que no se heredan. Ejemplo: los bordes, el color de fondo, etc. Las propiedades que si se heredan son las relacionadas a la tipografía, el tamaño, la familia, etc.

Como mencionamos, la propiedad background-color no es heredable. En el siguiente ejemplo veremos como heredar la propiedad. La propiedad font-size si se hereda, y veremos como al asignarla al elemento padre (div) el elemento hijo (span) la hereda. No es necesario utilizar el valor inherit ya que se hereda por defecto.

<div class="herencia-container">
  Texto en elemento div. <span>Ayer deseo, hoy realidad</span>
</div>
  
/* CSS */
.herencia-container {
  background-color: greenyellow;
  font-size: 25px;
}
.herencia-container span {
  background-color: pink;
}
Texto en elemento div. Ayer deseo, hoy realidad

Si queremos evitar la herencia de la propiedad font-size podemos asignar un nuevo valor o establacer el valor initial, que por defecto es de 16px.

<div class="herencia-container">
  Texto en elemento div. <span>Ayer deseo, hoy realidad</span>
</div>
  
/* CSS */
.herencia-container {
  background-color: greenyellow;
  font-size: 25px;
}
.herencia-container span {
  background-color: inherit;
  /* font-size: 10px; */
  font-size: initial;
}
Texto en elemento div. Ayer deseo, hoy realidad

Para saber qué propiedades son heredables podemos consultar propiedades heredables

Reseteo y Normalización de estilos

El navegador asigna ciertos estilos por default a las etiquetas. Aunque en la actualidad los distintos navegadores utilizan estilos iniciales similares, hace un tiempo tenían diferencias ocasionando que nuestra página se renderice de distinta manera según el navegador. Para eliminar esos estilos podemos utilizar un reseteo propio o utilizar algunas hojas de estilo predefinidas.

Los recursos más utlizados son:

  • Reset CSS. Resetea a cero algunos valores (margin, padding, etc) de etiquetas HTML.
  • Normalize CSS. A diferencia de reset css, normalize realiza una normalización de los valores. Todos los navegadores mostrarán la página con los mismos estilos iniciales pero no tiene porque ser "sin estilo".

Para utilizarlas podemos descargar el archivo .css (o crear un archivo y pegar el contenido) e invocarlo desde el html. Tenemos que realizar la invocación a través de la etiqueta Link antes de llamar a nuestros estilos para no sobreescribir nuestras definiciones. La cascada realizará su trabajo.

Si deseas definir los estilos desde cero para todos los elementos puedes utilizar Reset css, pero hay que tener en cuanta que resetea hasta las dimensiones de los encabezados. Por ello, es mejor utilizar herramientas como normalize porque unifica los estilos en los distintos navegadores pero no tendremos que definirlos desde cero.

Prefijos de los navegadores

HTML, CSS y JavaScript son los estándares web, no los tenemos que descargar a nuestros dispositivos para que funcionen. Son estándares que se encuentran en los navegadores web, que son las tecnologías que entienden. Por ello es que no tendremos que instalar nada, el navegador será capaz de interpretar estas tecnologías.

Los navegadores web son desarollados por distintas empresas (Chrome - Google, Firefox - Fundación Mozilla, Edge - Microsoft, Safari - Apple).

La W3C indica cuáles son los siguientes pasos en relación a los estándares HTML, CSS y JavaScript. Cuando una implementación aún no posee soporte al 100% pero se encuentra en etapa experimental podemos utilizar los prefijos para utilizarlas.

Prefijos de los navegadores

  • Microsoft: -ms-
  • Firefox: -moz-
  • Para el resto: -webkit-

Una herramienta que nos permite determinar los prefijos necesarios es autoprefixer

Cuando utilizamos herramientas como webpack que nos ayudan a empaquetar nuestra aplicación, automáticamente podemos minificar el código, agregar los prefijos, ofuscar el código, etc. Por eso no tenemos que hacernos demasiados problemas por los prefijos.
