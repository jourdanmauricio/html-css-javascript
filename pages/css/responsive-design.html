Contenedores Flexibles

Todos los elementos de nuestra interfaz tienen que ser flexibles y responsivos.

Cuando asignamos medidas absolutas y el elemento es mayor a la pantalla del dispositivo aparecerá la barra de scroll vertical.

<div class="box"></div>

/* CSS */
.box {
  width: 300px;
  height: 300px;
}

Una caja flexible posee las dimensiones definidas a través de unidades relativas. Si utilizamos la medida en porcentajes, al ancho relativo será con respecto a su contendor padre. Para utilizar altura en porcentajes tenemos que considerar que su contenedor padre tenga definidas la medida de altura (height).

Podemos utilizar el viewport como unidad de medida. En este caso, utilizamos el 20% de la altura de la pantalla.

<div class="box"></div>

/* CSS */
.box-flexible {
  width: 50%;
  height: 20vh;
}

Expresar el ancho en medidas relativas permite que el elemento se adapte (sea flexible) a los distintos dispositivos. En el caso anterior siempre ocupará el 50% del ancho.

También, existen propiedades CSS que nos permiten crear contenedores flexibles aunque utilicemos unidades absolutas como los pixeles.

Propiedades max-width, min-width, max-height y min-height

En lugar de definir una alto y ancho podemos definir máximos y mínimos. No definimos el ancho base, sino la máxima anchura que tomará el elemento. En el ejemplo el elemento no crecerá más allá de 960px y no decrecerá menos de 280px. A pesar de no utilizar unidades relativas conseguimos un contenedor flexible.

<div class="box"></div>

/* CSS */
.box-flexible {
  width: 50%;
  height: 20vh;
}

La altura siempre depende del contenido. En este caso, como el elemento no posee contenido toma la altura mínima (280px). Para verificar que la altura no supere los 480px creamos un nuevo contenedor con suficiente contenido para que supere la altura máxima.

Tamaños fijos VS Tamaños máximos y mínimos

Si en el contenedor definimos mínimos y máximos pero además definimos medidas width y height específicas en pixeles aplicará las medidas específicas ignorando los máximos y mínimos.

<div class="box-flexible-3">

/* CSS */
.box-flexible-3 {
  max-width: 960px;
  min-width: 280px;
  max-height: 480px;
  min-height: 280px;
  width: 300px;
  height: 300px;
}

Si definimos mínimos y máximos, y además width y height pero en unidades relativas como porcentajes o unidades viewport se aplicará el width y height pero se respetarán los mínimos y máximos.

<div class="box-flexible-4">

/* CSS */
.box-flexible-4 {
  max-width: 960px;
  min-width: 280px;
  max-height: 480px;
  min-height: 280px;
  width: 50%;
  height: 20vh;
}

Conclusión: las propiedades widht y height tienen mayor jerarquía que los mínimos y máximos. Si las unidades widht y height se expresan en unidades absolutas se ignorarán los mínimos y máximos. En cambio, si se expresan en unidades relativas serán consideradas.No utilizar máximos, mínimos y widht y height con unidades absolutas.

Multimedia Flexible

Los elementos multimedia es todo aquello que no sea texto:

  • img
  • audio
  • video
  • iframe
  • canvas
  • svg
  • picture
  • object
  • embed
  • map
  • area

Los contenidos multemedia deben ser flexibles y adaptarse al tamaño de su contendor. Lo logramos con la siguiente regla:

img, audio, video, iframe, canvas, svg, picture {
  max-width: 100%;
  height: auto;
}

Imágenes responsivas

Adaptamos la imagen al tamaño del contenedor. Al asignar la propiedad max-width al 100% no desbordadrá en su ancho, y al asignar auto en height logramos que la imagen no se deforme.

<img src="/assets/images/puesta-sol.jpg" alt="Puesta del sol">

/* CSS */
img {
  max-width: 100%;
  height: auto;
}

Etiqueta srcset y sizes

Estos atributos mejoran el rendimiento de la carga al colocar la misma imágen pero en diferentes tamaños y pesos dependiendo de la resolución de pantalla. En el ejemplo, se aplicaron filtros a los distintos tamaños para identificarlos visualmente, pero tendríamos que utilizar la misma imagen con distintos tamaños.

No es necesario incrustar imágenes grandes en la página si se está viendo en una pantalla móvil. Y, a la inversa, una imagen pequeña comienza a verse más pixelada cuando se muestra mayor que su tamaño original.

Algunos dispositivos tienen pantallas de alta resolución que necesitan imágenes más grandes de las que se espera para que luzcan bien.

Las imágenes vectoriales resolverían estos problemas, y lo hacen hasta cierto punto: son pequeñas en tamaño de archivo y escalan bien, y deberías usarlos siempre que sea posible.

Utilizando la resolución de pantalla (aspect ratio):

<img src="/assets/images/html5-3d-600.png"
srcset="/assets/images/html5-3d-600.png 1x, /assets/images/html5-3d-900.png 1.5x, /assets/images/html5-3d-1200.png 2x"
alt="HTML5">

Utilizando el ancho de la pantalla:

<img src="/assets/images/html5-3d-600.png"
srcset="/assets/images/html5-3d-600.png 600w, /assets/images/html5-3d-900.png 900w, /assets/images/html5-3d-1200.png 1200w"
alt="HTML5">

El atributo size nos permite trabajar con media queries dentro de las imágenes. En el siguiente ejemplo, cuando el tamaño de pantalla sea menor a 480px la imagen se renderizará en 300px, entre 480px y 600px se mostrará en 480px, y así sucesivamente 

<img src="/assets/images/html5-3d-600.png"
srcset="/assets/images/html5-3d-600.png 600w, /assets/images/html5-3d-900.png 900w, /assets/images/html5-3d-1200.png 1200w"
sizes="(max-width:480px) 300px, (max-width:600px) 480px, (max-width:768px) 600px, (max-width:1024px) 900px, (max-width:1200) 1200px"
alt="HTML5">

Mientras el atributo srcset nos permite cambiar la imágen a la misma de menor tamaño (mejora de rendimiento), el atributo sizes nos permite redimencionar un tamaño específico.

Etiqueta picture

El objetivo de esta etiqueta, no es solo mostrar diferentes tamaños de la misma imagen, sino también mostrar imágenes diferentes. Por ejemplo, recortar una imagen de forma diferente según el tamaño de la pantalla y las diferencias en el diseño. Esto se conoce como "dirección de arte".

Un ejemplo es el logotipo, en dispositivos pequeños se utiliza una versión minificada del logo, y en desktop se visualiza el logo con el nombre de la empresa.

<picture>
  <source srcset="/assets/images/kitten-large.png" media="(min-width:1024px)">
  <source srcset="/assets/images/kitten-medium.png" media="(min-width:600px)">
  <img src="/assets/images/kitten-small.png" alt="Kitten">
</picture>

Con la etiqueta picture también podemos aplicar otro tipo de cambios responsivos. Ejemplo: imágenes con formato webp que no son soportadas por algunos navegadores.

<picture>
<source srcset="/assets/images/butterfly.webp" type="image/webp">
<img src="/assets/images/butterfly.jpg" alt="Mariposa">
</picture>

Videos Responsivos

<video src="/assets/media/sunset_evening_deck_877.mp4" controls></video>

El video del ejemplo anterior posee un formato widescreen (16:9), pero si tuviera otro formato podemos hacerlo responsivo a través de una técnica que ya posee cierta antigüedad:

<div class="responsive-media">
  <video src="/assets/media/sunset_evening_deck_877.mp4" controls></video>
</div>

/* CSS */
.responsive-media {
  position: relative;
  width: 100%;
  height: 0;
  /*************************
   Fromato widescreen 16:9
   16 -> 100%
   9 -> 56.25%
  *************************/
  padding-bottom: 56.25%;
}

.responsive-media > * {
  position: absolute;
  width: 100%;
  height: 100%;
}

Finalmente, podemos obtener el mismo resultado colocando la resolución de pantalla o aspect ratio que deseemos con una nueva técnica en CSS:

<div class="aspect-ratio-16-9">
  <video src="/assets/media/sunset_evening_deck_877.mp4" controls></video>
</div>

/* CSS */
.aspect-ratio-16-9 {
  /* aspect-ratio: 4/3; */
  /* border: thick solid coral; */
  aspect-ratio: 16/9;
}

Media Queries

Desde la especificación de CSS 2.1, las hojas de estilo han tenido cierto grado de capacidad para el reconocimiento de dispositivos mediante el uso de tipos de medios. Por ejemplo: 

<link rel="stylesheet" media="print" href="print.css">

Cuando enviemos a imprimir se aplicarán los estilos definidos en el archivo print.css. Podría contener:

@page {
  size: A4;
}
      
body {
  font-size: 12pt;
  font-family: serif;
  background-color: #000;
  color: #fff;
}

Con CSS3, la W3C perfeccionó y mejoró los tipos de medios con características multimedia y con la capacidad de preguntar dichas características a los medios.

Esto no sólo hace posible inspeccionar el contenido que se entrega al dispositivo, sino también las características físicas reales del dispositivo.

El uso de media queries permite pedir fácilmente al navegador sus características, tales como anchura, altura, relación de aspecto, orientación, entre otras.

La sintaxis es la siguiente: 

@media screen and (max-width:480px) and (orientation:portrait) {
  /* 
  Código CSS que se aplicará cuando se cumpla la media queries
  */
}

Podemos aplicar los estilos de para impresión utilizando media queries CSS3. En nuestro archivo de estilos podemos agregar las reglas para ese medio. En los enlaces se encuentra la documentación para estilos de impresión.

@media print {
  @page {
    size: A5 landscape;
  }

  body {
    font-size: 10pt;
    font-family: sans-serif;
    background-color: #000;
    color: greenyellow;
  }
}

El navegador es capaz de interpretar la configuración y podemos actuar en consecuencia. Ejemplo, si el usuario posee configurada la reducción de movimiento en su navegador podríamos generar la siguiente media query donde indicamos que no aplique el desplazamiento suave en al documento html.

@media screen and (prefers-reduced-motion: reduce) {
  html {
    /* scroll-behavior: smooth; */
    scroll-behavior: auto;
  }
}
@media screen and (prefers-reduced-motion: no-preference) {
  html {
    scroll-behavior: auto;
    scroll-behavior: smooth;
  }
}

Otro tipo de media query popular es el relacionado al modo oscuro/claro en las páginas web. Dependiendo de las preferencias del usuario en su configuración del sistema operativo aplicará los estilos que definamos.

  @media screen and (prefers-color-scheme: dark) {
    html {
      background-color: black;
      color: teal;
    }
  }
  @media screen and (prefers-color-scheme: light) {
    html {
      background-color: white;
      color: darkblue;
    }
  }
  @media screen and (prefers-color-scheme: no-preference) {
    html {
      background-color: white;
      color: darkred;
    }
  }

Por último, veremos algunas media queries relacionadas a los tamaños de pantalla.

/* En Mobile First se usa min-width que significa lo mínimo, del valor que des, hacia arriba */
@media screen and (min-width: 480px) {
  html {
    background-color: lightpink;
  }
}
/* En Desktop First se usa max-width que significa lo máximo, del valor que des, hacia abajo */
@media screen and (max-width: 1024px) {
  html {
    background-color: lightgreen;
  }
}
@media screen and (min-width: 1200px) {
  html {
    background-color: white;
  }
}
Si comenzamos un desarrollo con mobile first deberíamos utilizar min-width. No se recimienda mezclar min-width y max-width.

Breakpoints

El breakpoint es esa medida en la que decimos realizar un cambio de estilo.

320px

Para dispositivos con pantallas pequeñas, como los teléfonos en modo vertical.
480px

Para dispositivos con pantallas pequeñas, como los teléfonos, en modo horizontal.
600px

Tabletas pequeñas, como el Amazon Kindle (600×800) y Barnes & Noble Nook (600×1024), en modo vertical.
768px

Tabletas de diez pulgadas como el iPad (768×1024), en modo vertical.
1024px

Tabletas como el iPad (1024×768), en modo horizontal, así como algunas pantallas de ordenador portátil, netbook, y de escritorio.
1200px

Para pantallas panorámicas, principalmente portátiles y de escritorio.

Con la tabla que Ethan Marcotte nos propone prácticamente están cubiertos teléfonos inteligentes, tabletas, phablets, laptops, netbooks, computadoras de escritorio y pantallas, tanto en modo vertical como horizontal, por lo que con estos 6 puntos de interrupción sólo tendríamos que validar un solo atributo en las media queries: el ancho.

De cualquier forma en el sitio screensiz.es, se puede estar revisando constantemente las características físicas con las que se fabrican los dispositivos.

¡Importante! Convierte tus Breakpoints a EMs

¿Por qué?

Aunque los pixeles se consideran una unidad de medida absoluta, en realidad son unidades relativas a la resolución de pantalla del dispositivo que lo visualiza, si dicho dispositivo tiene una densidad mayor a la normal, entonces la proporción de los pixeles cambiará, por ello es importante que los breakpoints de las media queries se conviertan a ems que si son unidades relativas y proporcionales.

Viewport

El viewport fue introducido por Apple en Safari móvil, para ayudar a mejorar la presentación de los sitios en dispositivos como el iPhone, iPod Touch o iPad; posteriormente se volvió un estándar en todos los navegadores.

La etiqueta viewport es el área visible de nuestro navegador y nos permite definir el ancho, alto y escala del área usada por el navegador para mostrar el contenido.

Si no agregamos esta meta etiqueta a nuestros documentos HTML tendremos problemas de visualización.

El viewport tiene 6 atributos importantes:

  1. width: define el ancho visible, suele usarse la constante device-width para que se adapte al ancho del dispositivo.
  2. height: define el alto visible, suele usarse la constante device-height para que se adapte al alto del dispositivo.
  3. initial-scale: define la escala inicial a la que deberá visualizarse el contenido, siendo 1 el 100%.
  4. minimum-scale: define la escala mínima a la que se podrá hacer zoom al contenido, por ejemplo si se configura .5 significa que se podrá hacer un zoom de hasta el 50%.
  5. maximum-scale: define la escala máxima a la que se podrá hacer zoom al contenido, por ejemplo si se configura 3 significa que se podrá hacer un zoom de hasta el 300%.
  6. user-scalable: define si el usuario puede escalar o no el contenido, sus valores son yes por defecto y no para bloquear esta propiedad.

De los 6 atributos mencionados, hay 2 que son muy importantes definir el ancho y la escala inicial, para que el contenido se adapte al tamaño del dispositivo.

Adicionalmente se recomienda que no se especifique la máxima escala, ni bloquear la capacidad de escalar el contenido, ya que independientemente de que el documento se diseñe con responsive, algunos usuarios con capacidades visuales diferentes, podrían tener problemas al ver el contenido y desearían aumentar el tamaño del contenido, si definimos una escala máxima o bloqueamos la capacidad de escalar el contenido produciremos en estos usuarios una mala y frustrante experiencia de usuario.

La propiedad del alto es opcional definirla, se utilizara cuando necesitemos condicionar el alto del contenido.

Por lo anterior la etiqueta meta viewport tendría que definirse de la siguiente manera: 

<meta name="viewport" content="width=device-width,initial-scale=1.0">

Grid Responsiva Artesanal con Flexbox

Vamos a desarrollar una maquetación en 12 columnas. Seleccionamos 12 columnas porque es muy divisible, podemos agregar 1, 2, 3, 4, 6 y 12 elementos. Utilizaremos Flexbox consguir la grilla.

Elementos del mismo espacio que podemos incluir en la grid.

  • 12 / 1 = 12
  • 12 / 2 = 6
  • 12 / 3 = 4
  • 12 / 4 = 3
  • 12 / 6 = 2
  • 12 / 12 = 1

La sumatoria de todos los elementos que queremos en una fila tiene que ser 12. Si es mayor, algún elemento pasará a la siguiente fila.

  • 5 + 7 = 12
  • 2 + 5 + 5 = 12
  • 4 + 2 + 3 + 3 =12

Para calcular que porcentaje le tenemos que asignar a cada columna realizamos una regla de 3 simple.

12 ----> 100%
?col ----> ???%

Tamaños (Mediaqueries)

  • xs - extrasmall - 0 - 479px - 480res
  • sm - small - 480px (30em) - 767px - 800res
  • md - medium - 768px (48em) - 991px - 1024res
  • lg - large - 992px (62em) - 1199px - 1280res
  • xl - extralarge -+1200px (75em) - >1281res

El viewport fue introducido por Apple en Safari móvil, para ayudar a mejorar la presentación de los sitios en dispositivos como el iPhone, iPod Touch o iPad; posteriormente se volvió un estándar en todos los navegadores.

La etiqueta viewport es el área visible de nuestro navegador y nos permite definir el ancho, alto y escala del área usada por el navegador para mostrar el contenido.

Feature Queries

Las Feature queries son reglas CSS que le preguntan al navegador si soportan o no una determinada regla o valor. Cuando se incorporan nuevas características y las queremos utilizar, podemos preguntar al navegador si se encuentra soportada e implementarlar o utilizar otras características.

Ejemplo, a partir del año 2012 podiamos maquetar en Flexblox pero en 2017 se incorporó la maquetación con Grid. Actualmente, todos los navegadores soportan Grid, pero inicialmente teníamos que definir una media query preguntanso si el navegador soportaba Grid para implementarlo. Si no lo soportaba maquetabamos con Flexbox.

Las feature queries nos permiten implementar nuevas características y mantener la compatibilidad hacia atrás a través del operador not. También podemos utilizar los operadores or y and como condicionales. 

/* Preguntamos si el navegador soporta la propiedad grid con el valor display */
@supports (display: grid) {
  /* 
  Código CSS que aplicará cuando cumpla con al feature query
  Clases para grid
  */
}
@supports not (display: grid) {
  /* 
  Código CSS que aplicará cuando cumpla con al feature query
  Clases para Flexbox
  */
}

@supports (display: grid) and (grid-template-columns: subgrid) {
  html {
    background-color: darkslateblue;
    color: lightseagreen;
  }
}
@supports (display: grid) or (grid-template-columns: subgrid) {
  html {
    background-color: white;
    color: black;
  }

Mobilegeddon

Es la actualización al algoritmo de Google, que está activo desde el 21 de abril de 2015.

Teniendo en cuenta el auge imparable de los dispositivos móviles y el impacto que éstos están teniendo en las costumbres de uso de la web, Google hizo un comunicado a través de su blog oficial comentando algunas de las novedades que trae la actualización de su algoritmo.

Mobilegeddon pretende actuar sobre todas aquellas webs que no sean amigables con los dispositivos móviles, es decir que no se adapten de forma óptima a dichos dispositivos.

Dado que las estrategias de Google se enfocan cada vez más hacia estos dispositivos, es normal que el buscador quiera asegurarse que las webs que indexa cumplen los mínimos de calidad para móviles.

Mobilegeddon, afecta:

  • Exclusivamente a las búsquedas realizadas desde dispositivos móviles.
  • La aplicación del algoritmo será en todos los idiomas y de forma global.
  • No se aplicará a sitios webs completos , sólo a las páginas que no cumplan los criterios.

Responsive Design

Es el paradigma actual del diseño web. Son un conjunto de buenas prácticas a nivel de código CSS para adaptar el contenido del documento HTML al tamaño de pantalla del dispositivo que lo visualice.

Tiene 3 principios:

  1. Grids Flexibles
  2. Imágenes Flexibles
  3. Media Queries

Fue propuesto por Ethan Marcotte en 2010.

Más información

  • Artículo Responsive Design
  • Libro Responsive Design

Desktop First VS Mobile First

Son las 2 estrategias que tiene el Responsive Design para adaptar el contenido al tamaño de la pantalla, centrándose en los dispositivos con pantalla más pequeña (Mobile First) o a la inversa, centrándose en los dispositivos con pantalla más grande (Desktop First).

Desktop First

(Graceful Degradation)

Las media queries se deben definir de la mayor a la menor.

Se utiliza la propiedad max-width para definir las mediaqueries.

Mobile First

(Progressive Enhancement)

Las media queries se deben definir de la menor a la mayor.

Se utiliza la propiedad min-width para definir las mediaqueries.

Utiliza programación JavaScript para los cambios en la interfaz.

Responsive Design + Server Side Components (RESS)

El Responsive Design + Server Side Components (RESS), trata de crear interfaces que se adapten a las capacidades y características de cada dispositivo (Hardware y Software).

Pero a diferencia del Adaptive Design, la programación que usa esta del lado del servidor. Las interfaces se crean desde el backend, lo que permite usar la misma vista (ruta) para todos los dispositivos.

Fluid Design

El Fluid Design es una técnica de Responsive Design que trata de escalar y adaptar el contenido al tamaño de la pantalla, sin la necesidad de ocupar media queries.

En lugar de diseñar para un determinado número de breakpoints, el Fluid Design crea un sistema dentro del cual los elementos escalan de manera proporcional y fluida.

Utiliza características y funciones modernas de CSS para hacer fluir las interfaces, por ejemplo:

  • Contenedores fluidos
  • Textos fluidos
  • Grids responsivas sin media queries
  • Uso de funciones como:
    • calc
    • min, max
    • clamp
  • Uso de unidades relativas como:
  • EMS
  • REMs
  • EXs
  • CHs
  • Porcentajes
  • Unidades del Viewport

Fue propuesto por Trys Mudford en 2020.
